# Nexus-Ark-Discord-Custom (0.2.9)

**元ネタはケノ氏作成の[Nexus Ark](https://github.com/kenomendako/Nexus-Ark)です。**

複数のAIペルソナをDiscordで運用することに特化しますた。  
本家が0.2.9にバージョンアップしたので追従するため、0.2.6.1で改造した内容の移植用リポジトリです。  

> **おそらく永遠にベータ版**

相変わらずインストール環境を丸っとリポジトリ管理してるので、デッドコピーになってるのは自分でもどうかと思う。  


## 改造に至った動機

拙作の[Nexus Ark Multi Discord Custom 0.2.6.1版](https://github.com/Xen-Shuto/Nexus-Ark-Discord-Custom_0.2.6_old)でやっている改造内容をケノ氏に共有したところ、やりたいことの方向性がほぼ同じということで、ありがたいことに大部分を本家で実装していただいたのだが、そこはそれ、細かいところが気になったので独自にブラッシュアップすることにしたのである。  
  
0.2.8に実装してたら0.2.9がリリースされたの対応した。  


## やりたいことと、その進捗（改造0.2.6.1と0.2.8との比較を0.2.9に移植）

1. ★移植完了★【0.2.8実装済み】外部連携のTwitter設定とDiscord Bot設定の不具合修正  
　　　⇒ルーム別の設定になっているがルーム切り替え時に不具合があったけど、本家はルーム設定を呼び出すボタン実装で解消。  
　　　　ルーム切り替え時に設定内容を空で表示することで、ルーム切り替え後に誤って設定を上書き保存できないようにする。
  
2. ★移植完了★【0.2.8未実装】画像生成用APIにローカルを追加する。  
  
3. ★移植完了★【0.2.8未実装】Twitter設定に、「引用、リプライは承認が必要」を追加する。  
　　　⇒情報共有後に追加した機能なので・・・。  
  
4. ☆破棄☆【0.2.8実装済み】自律行動時にwebhookを使わずにdiscordのデフォルトチャンネルにメッセージや画像を送れるようにする。  
  
5. ★移植完了★【0.2.8未実装】Discordへの送信時、画像パスを本文に検知したら画像に置き換える。  
　　　⇒情報共有後に追加した機能なので・・・。  
  
6. ★移植完了★【0.2.8未実装？】Twitterの下書きをDiscordから承認できるようにする。  
　　　⇒情報共有済機能だが、承認/否認方法まで共有できていない。（0.2.8はコマンド制？）  
　　　　・ボタン構築機能を使用した承認/否認方法に変更する。  
　　　　・非同期でのポストを行い、Botを止めない。  
　　　　・Nexus Arkを再起動しても操作可能にする。  
  
7. ★移植完了★【独自】APIローテーション方式の見直し  
　　　⇒Google神に怒られたので……。  
　　　　「過剰にお行儀の良い」新方式を実装（１つのキーを使い潰してからローテーション）。  
　　　　RPM待機(65-75s)、RPDの25時間ロック、有料キー後の無料キー復帰ロジックを統合。  
  
8. ★移植完了★【独自】RAG更新回りのロジックの見直し  
　　　⇒上記7でGoogle神に怒られ原因……。  
　　　　0.2.6.1で重複してたロジックとかウェイト無しでループ回してたとか……。  
  
9. ☆破棄☆【0.2.8実装済み】Discordのイベントハンドラのマルチスレッド化  
　　　⇒情報共有済機能。
　　　　応答受付はマルチ、応答そのものはシングルとすることで、AIペルソナの一貫性が保たれる。
  
10. ▽未移植▽【0.2.8実装済み？】ペルソナ側から権限のある任意のチャンネルにメッセージや画像を送れるようにする。  
　　　⇒情報共有済機能。おま環？　挙動がちょっと怪しい。自律行動含む。  
  
11. ▽未移植▽【0.2.8実装済み？】Discordで複数のBotが存在するチャンネルでメンション及び名前で呼んだ際、呼ばれたBotだけが反応するようにする。  
　　　⇒情報共有済機能。おま環？　挙動がちょっと怪しい。  
　　　　ついでにリアクションにも反応させる。
  
999.▽未対応▽【0.2.8実装済み】自律行動時にbot同士で会話（３巡の制限）  
　　　⇒３巡の制限を解除し、自然な形で会話が終わるようにする。
